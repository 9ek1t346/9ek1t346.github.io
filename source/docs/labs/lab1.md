# Лабораторная работа №1

## Тема
Создание и развертывание статического сайта на базе MkDocs с публикацией на GitHub Pages.

## Цель работы
Освоить процесс создания статического сайта с использованием MkDocs, научиться организовывать структуру документации проекта и публиковать готовый сайт через GitHub Pages.

## Что требовалось выполнить
В рамках лабораторной работы необходимо было:
1. создать публичный репозиторий на GitHub;
2. настроить GitHub Pages для публикации из каталога `/docs` ветки `main`;
3. клонировать репозиторий на локальный компьютер;
4. создать и активировать виртуальное окружение Python;
5. установить MkDocs;
6. создать структуру сайта;
7. настроить файл `mkdocs.yml`;
8. подготовить не менее 5 страниц сайта;
9. выполнить локальную проверку сайта;
10. собрать сайт и подготовить его к публикации.

## Ход выполнения

### 1. Создание репозитория
Был создан публичный репозиторий `9ek1t346.github.io`.  
Такое имя позволяет публиковать сайт по адресу вида `https://9ek1t346.github.io/`.

### 2. Подготовка окружения
Репозиторий был клонирован на локальный компьютер.  
После этого было создано виртуальное окружение `.venv`, чтобы изолировать зависимости проекта.

### 3. Установка MkDocs
В виртуальное окружение были установлены:
- `mkdocs`
- `mkdocs-material`

В качестве темы оформления была выбрана **Material for MkDocs**, так как она обеспечивает удобную навигацию, современный внешний вид и хорошо подходит для учебного портфолио.

### 4. Создание структуры сайта
С помощью команды `mkdocs new source` была создана базовая структура проекта.  
После этого были добавлены страницы:
- главная страница;
- страница «Об авторе»;
- раздел «Лабораторные работы»;
- страница с отчётом по лабораторной работе №1;
- страница с шаблоном отчёта.

### 5. Настройка конфигурации
В файле `source/mkdocs.yml` были настроены:
- название сайта;
- тема оформления;
- язык интерфейса;
- структура навигации через `nav`;
- сборка сайта в корневой каталог `docs`.

### 6. Локальная проверка
После настройки страниц сайт был проверен локально с помощью встроенного сервера MkDocs.  
Проверка показала, что страницы открываются корректно, меню работает, а структура сайта соответствует требованиям.

### 7. Сборка и подготовка к публикации
После локальной проверки сайт был собран в корневой каталог `docs`, который используется GitHub Pages как источник публикации.  
Также был добавлен файл `.nojekyll` для корректной публикации статического сайта.

## Использованные команды

```bash
git clone https://github.com/9ek1t346/9ek1t346.github.io.git
cd 9ek1t346.github.io

python3 -m venv .venv
source .venv/bin/activate

python -m pip install --upgrade pip
python -m pip install mkdocs mkdocs-material

mkdocs new source

python -m mkdocs serve -f source/mkdocs.yml
python -m mkdocs build -f source/mkdocs.yml

# Шаблон отчёта

## Тема
Создание и развёртывание статического сайта на базе MkDocs с публикацией на GitHub Pages.

## Цель работы
Освоить процесс создания статического сайта с использованием генератора документации MkDocs, научиться организовывать структуру страниц проекта и подготовить сайт к публикации через GitHub Pages.

## Задание
В рамках лабораторной работы необходимо было:
1. создать публичный репозиторий на GitHub;
2. настроить локальное окружение для работы с MkDocs;
3. создать структуру сайта;
4. оформить несколько страниц;
5. настроить навигацию через файл `mkdocs.yml`;
6. проверить работу сайта локально;
7. подготовить сайт к публикации.

## Ход выполнения

### 1. Создание репозитория
Сначала был создан публичный репозиторий на GitHub с именем `9ek1t346.github.io`.  
Такое имя выбрано потому, что оно соответствует формату пользовательского сайта GitHub Pages и позволяет публиковать сайт по адресу `https://9ek1t346.github.io/`.

После этого репозиторий был клонирован на локальный компьютер, и дальнейшая работа выполнялась в локальной копии проекта через PyCharm и терминал.

### 2. Подготовка локального окружения
Для изоляции зависимостей было создано виртуальное окружение Python `.venv`.  
Это позволило устанавливать необходимые пакеты отдельно от системного Python.

После создания окружение было активировано, а затем в него были установлены:
- `mkdocs`;
- `mkdocs-material`.

В качестве темы оформления была выбрана **Material for MkDocs**, так как она имеет современный внешний вид, удобную навигацию и подходит для учебного сайта-портфолио.

### 3. Создание структуры проекта
С помощью команды `mkdocs new source` был создан базовый каркас проекта MkDocs.  
После этого в репозитории появилась папка `source`, внутри которой расположены:
- каталог `docs` для страниц сайта;
- файл `mkdocs.yml` для настройки сайта.

Затем структура была дополнена вручную: были созданы отдельные страницы и подпапки для организации содержимого.

### 4. Создание страниц сайта
Чтобы сайт соответствовал требованиям лабораторной работы, была сформирована структура минимум из пяти страниц:

- главная страница;
- страница «Об авторе»;
- страница раздела «Лабораторные работы»;
- страница с отчётом по лабораторной работе №1;
- страница с шаблоном отчёта.

Для этого были созданы следующие файлы:
- `source/docs/index.md`
- `source/docs/about.md`
- `source/docs/labs/index.md`
- `source/docs/labs/lab1.md`
- `source/docs/labs/template.md`

Такой подход позволил сделать структуру сайта логичной и иерархической.

### 5. Заполнение содержимого страниц
После создания файлов каждая страница была заполнена текстом.

На главной странице было размещено краткое описание сайта и его назначения.  
На странице «Об авторе» были указаны сведения об авторе и цель создания сайта.  
В разделе «Лабораторные работы» был оформлен список доступных материалов.  
На странице лабораторной работы №1 был размещён сам отчёт по выполнению задания.  
На странице шаблона отчёта была оформлена структура, которую можно использовать для следующих лабораторных работ.

### 6. Настройка конфигурации MkDocs
Далее был настроен файл `source/mkdocs.yml`.

В нём были указаны:
- название сайта;
- описание сайта;
- автор;
- тема оформления;
- язык интерфейса;
- структура навигации через параметр `nav`.

Навигация была организована так, чтобы пользователь мог перейти:
- на главную страницу;
- на страницу «Об авторе»;
- в раздел «Лабораторные работы»;
- к странице лабораторной работы №1;
- к странице шаблона отчёта.

### 7. Создание служебных файлов
Дополнительно были созданы служебные файлы проекта:
- `.gitignore`
- `README.md`

Файл `.gitignore` был нужен для исключения из Git виртуального окружения, служебных Python-файлов и системных файлов macOS.

В `README.md` было кратко описано назначение проекта, добавлены ссылки на репозиторий и сайт, а также приведено обоснование выбора темы оформления.

### 8. Проверка сайта локально
После настройки структуры и конфигурации сайт был запущен локально командой запуска встроенного сервера MkDocs.

Локальная проверка позволила убедиться, что:
- страницы открываются без ошибок;
- навигация работает корректно;
- структура меню соответствует требованиям;
- содержимое отображается правильно.

Во время настройки были обнаружены и исправлены проблемы с рабочей директорией и виртуальным окружением: команды сначала выполнялись не из корня репозитория, а также использовалось не то окружение Python. После переключения в правильную папку проекта и установки пакетов в нужное `.venv` сайт начал запускаться корректно.

### 9. Подготовка к публикации
После локальной проверки проект был подготовлен к сборке сайта в каталог `docs` корня репозитория и публикации через GitHub Pages.

На данном этапе была сформирована вся необходимая структура проекта:
- репозиторий;
- страницы сайта;
- конфигурация MkDocs;
- навигация;
- служебные файлы;
- локально работающая версия сайта.

## Использованные команды

```bash
git clone https://github.com/9ek1t346/9ek1t346.github.io.git
cd 9ek1t346.github.io

python3 -m venv .venv
source .venv/bin/activate

python -m pip install --upgrade pip
python -m pip install mkdocs mkdocs-material

mkdocs new source

python -m mkdocs serve -f source/mkdocs.yml
python -m mkdocs build -f source/mkdocs.yml
